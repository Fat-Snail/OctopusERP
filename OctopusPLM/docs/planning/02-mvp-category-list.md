# OctopusPLM MVP 类目清单（对齐 1688 商品管理）

## 1. MVP 原则

1. 优先覆盖高频业务和规则复杂度适中的类目。
2. 每个类目都要覆盖：基础属性、销售属性、履约属性、详情组件。
3. 类目配置上线必须不依赖代码发布。

## 2. MVP 类目范围（首批 5 个）

1. `智能手机`
2. `笔记本电脑`
3. `T恤`
4. `连衣裙`
5. `休闲零食`

备注：以上类目与当前 `PlmDbSeeder` 已有样例一致，便于低风险切换。

## 3. 类目配置清单（样板）

### 3.1 智能手机

1. 基础属性：品牌、型号、上市时间、机身颜色、售后政策。
2. 销售属性（SKU 轴）：存储容量、内存。
3. 关键技术属性：CPU、屏幕尺寸、电池容量、充电功率、网络制式、NFC。
4. 详情组件：主图轮播、卖点图文、参数表、包装清单、售后说明、FAQ。
5. 规则：存储容量和内存必选；5G 机型必须填写网络频段；电池容量必须为正整数。

### 3.2 笔记本电脑

1. 基础属性：品牌、系列、型号、操作系统、保修年限。
2. 销售属性（SKU 轴）：内存、硬盘容量、颜色。
3. 技术属性：CPU 型号、显卡型号、屏幕尺寸、分辨率、重量、接口类型。
4. 详情组件：场景图、参数对照表、性能说明、包装清单、质保说明。
5. 规则：带独显时显存字段必填；重量单位统一 `kg`，保留 2 位小数。

### 3.3 T恤

1. 基础属性：品牌、版型、领型、袖长、面料成分、执行标准。
2. 销售属性（SKU 轴）：颜色、尺码。
3. 物流属性：单件重量、包装尺寸、发货时效。
4. 详情组件：主图、尺码表、材质说明、洗护说明、模特展示、退换货说明。
5. 规则：尺码表组件必填；面料成分需命中词典（棉/涤纶/氨纶等）。

### 3.4 连衣裙

1. 基础属性：品牌、裙长、腰型、领型、图案、面料成分。
2. 销售属性（SKU 轴）：颜色、尺码。
3. 物流属性：重量、包装体积。
4. 详情组件：主图、穿搭图集、尺码建议、面料细节、洗护说明。
5. 规则：连衣裙必须配置至少 1 张“上身图”；尺码与胸围字段联动校验。

### 3.5 休闲零食

1. 基础属性：品牌、品名、产地、配料表、保质期、储存条件。
2. 销售属性（SKU 轴）：口味、规格净含量。
3. 合规属性：生产许可证编号、执行标准、过敏原提示。
4. 详情组件：主图、配料图、营养成分表、食用建议、资质证照、FAQ。
5. 规则：食品类必须上传资质组件；保质期与生产日期逻辑校验。

## 4. 详情组件库（MVP 6+2）

1. 标准组件（首发 6 个）：`主图轮播`、`图文卖点`、`参数表`、`尺码/规格表`、`资质证照`、`FAQ`。
2. 预备组件（灰度 2 个）：`视频模块`、`对比模块`。

## 5. 配置与治理要求

1. 类目配置必须带版本号和生效时间。
2. 每次变更自动生成差异记录（字段新增/删减/规则变化）。
3. 上线前走“配置校验 + 沙箱预览 + 审核发布”。

## 6. MVP 成功标准

1. 新增类目配置在 `1 天内` 上线，不需要研发发版。
2. 商品编辑首轮通过率提升到 `85%+`（减少驳回）。
3. 重点类目商品发布时长降低 `30%+`。
4. 详情页缺失必填模块的比例低于 `2%`。
